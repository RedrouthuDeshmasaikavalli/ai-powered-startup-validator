[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's trace how a request flows through the architecture.
First, the input enters the START node and lands at the security_checkpoint.
This checkpoint scrubs personal data and filters out disallowed business models, routing bad ideas directly to the security_event node.
Clean inputs route to the lead orchestrator agent.
The orchestrator delegates to two specialized sub-agents: the market_analyst and the financial_advisor.
These sub-agents pull fresh data on competitors, TAM, and margins from our local Model Context Protocol server.
The orchestrator then compiles an initial assessment and sends it to the hitl_gate.
Here, the validation pauses to ask the human user for feedback or clarifications.
Once the user confirms, the final_evaluator takes over to produce a structured JSON viability report, which is converted to a clean report by the render_output node.
